## Server-Side Rendering and Static Site Generation

One of the most exciting features of Next.js is its **powerful rendering options**. Unlike traditional React apps that rely on client-side rendering (CSR), Next.js lets you choose between **server-side rendering (SSR)** and **static site generation (SSG)** based on your needs.

- **SSR (Server-Side Rendering)**: The page is generated on the server at request time, ensuring fresh data on every load. This is ideal for dynamic content like user dashboards and frequently updated pages.
- **SSG (Static Site Generation)**: The page is pre-rendered at build time, making it lightning-fast. This is perfect for blogs, marketing pages, and content that doesnâ€™t change often.

To implement SSR, use `getServerSideProps` in your page component:

```js
export async function getServerSideProps() {
  const res = await fetch("https://api.example.com/data");
  const data = await res.json();
  return { props: { data } };
}
```

For SSG, use `getStaticProps` instead:

```js
export async function getStaticProps() {
  const res = await fetch("https://api.example.com/data");
  const data = await res.json();
  return { props: { data } };
}
```
